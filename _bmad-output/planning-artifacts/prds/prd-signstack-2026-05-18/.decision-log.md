**Context:** Open question from brief. User decided: three options for MVP 1.

### D-004 — Legal disclaimer resolved
**Context:** Open question from brief. User decided yes.
**Decision:** Both the product UI and the README will include a short disclaimer that SignStack adds visual signatures and does not create cryptographic certificate-based digital signatures.
**Status:** Applied. See FR-18.

### D-005 — Scope discipline confirmed
**Context:** User explicitly instructed PRD to preserve brief scope discipline.
**Decision:** MVP 1 = visual signing only. Phase 2 = fill-out overlays. Phase 3 = combine PDFs. Auth, DB, cloud, email, payments, crypto signing, AcroForm, AI are non-goals.
**Status:** Applied.

### D-006 — Single document per session (MVP 1)
**Context:** No multi-PDF upload in scope; natural consequence is one document per session.
**Decision:** [ASSUMPTION] MVP 1 is single-session, single-document, stateless. Starting over with a new document requires a page refresh. Acceptable for MVP 1.
**Status:** Applied. Flagged as A-2 in Assumptions Index.

### D-007 — Signature reuse within session
**Context:** UX question — can a user reuse the same signature across multiple placements?
**Decision:** [ASSUMPTION] Signature created once per session; user can place it on multiple pages without recreating it. Implementation detail for architecture. Flagged as OQ-1 for UX clarification.
**Status:** Applied.

### D-008 — File size limit
**Context:** Need a bound for PRD.
**Decision:** [ASSUMPTION] 25 MB hard limit for MVP 1. Needs validation against pdf-lib memory constraints during architecture phase. Flagged as OQ-3.
**Status:** Applied.

### D-009 — Touch/stylus input
**Context:** Drawing canvas implementation question.
**Decision:** [ASSUMPTION] Touch/stylus input supported incidentally on the Signature Canvas (react-signature-canvas handles it). Not a primary target; mobile-optimized layout not in MVP 1 scope. Flagged as OQ-2.
**Status:** Applied.

### D-010 — Disclaimer placement
**Context:** Where in the UI does the disclaimer appear?
**Decision:** [ASSUMPTION] Persistent inline text near export button + README. Not a modal for MVP 1. Flagged as OQ-4 for UX decision.
**Status:** Applied.

### D-013 — MVP 1 deployment clarification
**Context:** User clarified before UX design phase.
**Decision:** MVP 1 is a locally running proof of concept. CDN, cloud hosting, Vercel/Cloudflare Pages, and any infrastructure considerations are future production-readiness concerns, not MVP 1 implementation requirements. The app runs locally via Next.js (`next dev`). All browser-side PDF operations remain unchanged.
**Status:** Applied. PRD Platform section language about CDN/deployment is informational forward-looking context only — not an MVP 1 deliverable.
