# Decision Log — SignStack Product Brief

## 2026-05-18

### D-001 — Fast Path selected
**Context:** User provided comprehensive product direction upfront.
**Decision:** Draft brief from provided input rather than using Coaching path. Assumptions flagged inline.
**Status:** Applied.

### D-002 — MVP 1 scope locked to visual signing only
**Context:** User explicitly scoped MVP 1 to PDF upload, page rendering, drawn/typed signature creation, drag/resize overlay, and export.
**Decision:** No text fill-out, no combining, no auth, no cloud in MVP 1.
**Status:** Applied.

### D-003 — Visual signing defined as image/text embedding, not cryptographic
**Context:** User explicitly distinguished visual signing from cryptographic certificate-based digital signatures.
**Decision:** "Visual signing" = embedding a visible signature artifact (drawn image or typed text rendered as image) into the PDF. No PKI, no certificate chain, no compliance-grade signing.
**Status:** Applied. Noted as Non-Goal.

### D-004 — Text fill-out defined as visual overlays, not AcroForm editing
**Context:** User explicitly distinguished Phase 2 fill-out from native PDF AcroForm field editing.
**Decision:** Phase 2 overlays are rendered onto the PDF during export, not wired to existing form fields.
**Status:** Applied. Noted as Non-Goal.

### D-005 — Local-first processing as core privacy differentiator
**Context:** User stated local/browser-first processing as a key constraint.
**Decision:** All PDF processing runs client-side in the browser. No document upload to servers. This is a design constraint and a marketing differentiator.
**Status:** Applied.

### D-006 — No auth, DB, cloud storage, email, payments in MVP 1
**Context:** User listed explicit exclusions for MVP 1.
**Decision:** All listed items are Non-Goals for MVP 1.
**Status:** Applied.
